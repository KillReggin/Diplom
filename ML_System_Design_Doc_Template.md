# ML System Design Doc - [RU]
## Дизайн ML системы - \<Fruitinspect\> \<MVP\> \<1\>

### 1. Цели и предпосылки 
#### 1.1. Зачем идем в разработку продукта?  

FruitInspect — веб-портал SaaS по подписке для прогнозирования урожайности виноградников, где клиенты ( винодельни ) загружают видео с плантаций, модель детектирует гроздья и прогнозирует урожай по весу. Бизнес-цель — помочь винодельням планировать закупки и найм рабочих заранее на сезон. ML повышает точность подсчета до 85% против ручных методов. Успех MVP — запуск портала с 10+ клиентами. Оборот по издержкам для средней винодельни составляет от 25 до 35 млн рублей за сезон. Совокупная потенциальная экономия: 10-15 млн руб/сезон.
Ошибки закупки сырья до 16 млн руб/сезон. После внедрения -40-50%.
Переплата сезонным рабочим 4 млн руб/сезон. После внедрения -50%.
Простой переработки 8 млн руб/ сезон. После внедрения -30%
Избыточная аренда тары 1.5 млн руб/ сезон. После внедрения – 50%.


#### 1.2. Бизнес-требования и ограничения  

Бизнес-требования: детекция гроздей (mAP>0.85), прогноз веса с MAE<10%; тарифы Starter (10 видео/мес, 15k руб), Pro (30 видео, 30k руб), VIP (неограничено, 70k+ руб) с интеграцией ERP. Ограничения: только RGB-видео, фокус на винодельнях РФ
#### 1.3. Что входит в скоуп проекта/итерации, что не входит   

В MVP входит: портал с регистрацией/загрузкой видео, YOLO-модель детекции, подсчет веса, отчеты. Не входит: интеграция ERP, API для third-party

#### 1.4. Предпосылки решения  

Данные: видео виноградников (RGB); выход — прогноз урожая (тонн/га). Горизонт — текущий урожай, гранулярность — по видео
### 2. Методология `Data Scientist`     

#### 2.1. Постановка задачи  

Задача — object detection гроздей на видео + регрессия веса по bounding boxes и размеру.

#### 2.2. Блок-схема решения  

Регистрация/подписка -> Загрузка видео (S3) -> Extract frames (OpenCV) -> YOLOv8+ (детекция) -> CNN вес (по размеру) -> Отчет (PDF/дашборд)
Бейзлайн: OpenCV contours; MVP: fine-tuned YOLO.
