# ML System Design Doc - [RU]
## Дизайн ML системы - \<Fruitinspect\> \<MVP\> \<1\>

### 1. Цели и предпосылки 
#### 1.1. Зачем идем в разработку продукта?  
Проблема: Винодельни не могут точно спрогнозировать урожай до сбора — переоценка приводит к убыткам на закупку сырья (+30% расходов), недооценка — к простою рабочих (20% холостого времени). Текущие методы (визуальный осмотр) дают точность 60-70%, сильно субъективны.
Ценность ML: SaaS-портал FruitInspect анализирует видео с виноградника → прогноз урожая с точностью 85% → ROI 2.5x за счет оптимизации закупок и найма
TAM/SAM/SOM: TAM 5 млрд руб (виноделие РФ+ЕС), SAM 300 млн руб (цифровизация), SOM 50 млн руб (50 средних виноделен).

1.7. Бизнес-требования и ограничения
Функциональные: RGB-видео → детекция → расчет веса → отчет → email-уведомление. Нефункциональные: Uptime 99.5%, <5 мин/видео, масштабирование до 200 видео/день. Бюджет: 1.2 млн руб (Разработка 600k, GPU 300k, Продажи 300k). Критерии выхода: 5 платящих клиентов, точность 85%+.
#### 1.3. Что входит в скоуп проекта/итерации, что не входит   

В MVP входит: портал с регистрацией/загрузкой видео, YOLO-модель детекции, подсчет веса, отчеты. Не входит: интеграция ERP, API для third-party

#### 1.4. Предпосылки решения  

Данные: видео виноградников (RGB); выход — прогноз урожая (тонн/га). Горизонт — текущий урожай, гранулярность — по видео
### 2. Методология `Data Scientist`     

#### 2.1. Постановка задачи  

Задача — object detection гроздей на видео + регрессия веса по bounding boxes и размеру.

#### 2.2. Блок-схема решения  

Регистрация/подписка -> Загрузка видео (S3) -> Extract frames (OpenCV) -> YOLOv8+ (детекция) -> CNN вес (по размеру) -> Отчет (PDF/дашборд)
Бейзлайн: OpenCV contours; MVP: fine-tuned YOLO.
